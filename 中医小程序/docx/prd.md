# 基于中医症状文本/舌象/面相的智能问答小程序 - 产品需求文档（PRD）

**版本**：V2.0  
**文档状态**：正式版  
**撰写日期**：2025-11-22  
**核心特性**：接入智能体 API 实现智能问答

---

## 一、项目背景与目标

### 1.1 项目背景

中医问诊依赖「望闻问切」，普通用户难以获得便捷的中医健康指导。随着 AI 大模型技术的成熟，通过接入智能体 API，结合症状文本、舌象、面相等多模态信息，可以构建一个智能化的中医健康问答工具。

### 1.2 项目目标

- 开发一款基于智能体 API 的中医健康问答小程序
- 支持症状文本问答、舌象识别、面相识别、体质评估
- 提供个性化的健康调理建议和就医指导
- 坚持健康科普定位，不替代专业医师诊断

### 1.3 核心价值

- **智能化**：接入大模型 API，提供专业的中医分析
- **多模态**：融合文本、图像等多种信息源
- **便捷性**：小程序形态，随时随地使用
- **个性化**：基于用户体质提供定制化建议

## 二、技术架构设计

### 2.1 整体架构

```
小程序前端 (uni-app)
    ↓
后端服务层 (Node.js/Java)
    ↓
```

### 2.2 智能体 API 选型

**推荐平台**：
- 百度千帆（文心一言）
- 阿里通义千问
- 腾讯混元
- 字节豆包

**核心能力要求**：
1. 多轮对话能力
2. 图像理解能力（Vision API）
3. 知识增强能力（RAG）
4. 结构化输出
5. 内容安全审核

## 三、功能模块设计

### 3.1 用户登录模块

**登录方式**：
- 微信授权登录（主要）

**功能要求**：
- 登录后才能使用核心功能
- 支持历史记录保存
- 数据加密存储

### 3.2 症状文本问答

**输入方式**：
- 自由文本输入
- 常见症状标签快选

**API 调用流程**：
```
1. 用户输入症状
2. 构建 Prompt（角色+症状+要求）
3. 调用智能体 Chat API
4. 解析结构化结果
5. 展示分析报告
```

**输出内容**：
- 体质类型分析
- 症状解释
- 调理建议（饮食/运动/作息）
- 就医建议
- 免责声明

### 3.3 舌象识别

**采集流程**：
- 拍照指导（光线/角度）
- 图片质量检测
- 上传到服务器

**API 调用**：
```
调用 Vision API
- 输入：舌象图片 + 分析要求
- 输出：舌质/舌苔特征 + 体质倾向
```

**分析维度**：
- 舌质：颜色、形态
- 舌苔：颜色、厚薄、润燥
- 体质倾向判断

### 3.4 面相识别

**采集要求**：
- 正脸、无遮挡
- 光线均匀

**分析内容**：
- 面色（红/黄/白/黯）
- 局部特征（眼周/唇色）
- 健康状态评估

### 3.5 体质评估

**评估方式**：
- 标准化问卷（9 种体质）
- 结合舌象/面相结果

**输出结果**：
- 主要体质类型
- 次要体质类型
- 调养建议

### 3.6 历史记录

- 咨询时间
- 问题摘要
- 分析结果
- 支持删除

## 四、智能体 API 接入详细方案

### 4.1 Prompt 工程设计

**System Prompt 模板**：
```
你是一位中医健康顾问，擅长体质辨识和健康指导。

【角色定位】
- 提供健康科普和生活建议
- 不诊断疾病，不开处方
- 使用通俗语言

【输出格式】JSON
{
  "constitution": "体质类型",
  "score": 85,
  "symptom_analysis": "症状分析",
  "lifestyle_advice": {
    "diet": ["建议1", "建议2"],
    "exercise": ["建议1"],
    "sleep": ["建议1"]
  },
  "medical_advice": {
    "need_medical": false,
    "urgency": "low",
    "reason": "说明"
  }
}
```

### 4.2 API 调用示例

**文本问答**：
```javascript
const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'gpt-4',
    messages: [
      {role: 'system', content: SYSTEM_PROMPT},
      {role: 'user', content: userSymptoms}
    ],
    temperature: 0.7,
    max_tokens: 2000
  })
});
```

**图像识别**：
```javascript
const response = await fetch('https://api.openai.com/v1/chat/completions', {
  method: 'POST',
  body: JSON.stringify({
    model: 'gpt-4-vision-preview',
    messages: [{
      role: 'user',
      content: [
        {type: 'text', text: '请分析这张舌象图片'},
        {type: 'image_url', image_url: {url: imageBase64}}
      ]
    }]
  })
});
```

### 4.3 知识库构建

**内容结构**：
1. 9 种体质详细描述
2. 症状对照表
3. 调理方案库
4. 中医理论知识

**技术实现**：
- 向量数据库：Pinecone / Milvus
- Embedding 模型：text-embedding-ada-002
- 检索策略：语义检索 + 关键词匹配

### 4.4 成本控制

**成本估算**（以 GPT-4 为例）：
- 文本问答：$0.03-0.06 / 次
- 图像识别：$0.01-0.02 / 张
- 月活 1000 用户：约 $300-600 / 月

**优化策略**：
1. 缓存相似问题答案
2. 简单问题使用 GPT-3.5
3. 限制对话轮次
4. Token 优化

## 六、安全与合规

### 6.1 医疗合规

- 避免"治愈"、"确诊"等绝对化表述
- 使用"可能"、"倾向"、"建议"等词汇
- 明确免责声明
- 急重症必须建议就医

### 6.2 数据安全

**传输安全**：
- 全程 HTTPS 加密
- API Key 服务端存储
- 敏感信息脱敏

**存储安全**：
- 健康数据加密存储
- 图片使用私有 OSS
- 定期清理过期数据（>90天）

**API 安全**：
- 请求签名验证
- 频率限制（防刷）
- IP 白名单

### 6.3 隐私保护

- 不存储原始图片
- 用户可删除历史记录
- 提供数据导出功能
- 账号注销能力

## 十、风险与挑战

### 10.1 技术风险

- API 稳定性依赖第三方
- 成本可能超预算
- 识别准确度有限

**应对措施**：
- 多平台备份
- 成本监控告警
- 人工审核机制

### 10.2 合规风险

- 医疗广告法限制
- 用户隐私保护
- 内容安全审核

**应对措施**：
- 法务审核文案
- 完善隐私政策
- 接入内容审核 API

### 10.3 运营风险

- 用户留存率低
- 问答质量不稳定
- 用户反馈处理

**应对措施**：
- 优化用户体验
- 持续优化 Prompt
- 建立反馈机制

### 12.1 中医体质分类

1. **平和体质**：阴阳气血调和
2. **气虚体质**：疲倦乏力、气短懒言
3. **阳虚体质**：怕冷、手脚冰凉
4. **阴虚体质**：口干、五心烦热
5. **痰湿体质**：体型肥胖、痰多
6. **湿热体质**：面部油腻、口苦
7. **血瘀体质**：面色晦暗、易长斑
8. **气郁体质**：情绪低落、胸闷
9. **特禀体质**：过敏体质

**头部**：头痛、头晕、失眠、多梦
